We have a classic Electron "Native Module" conflict.

The error NODE_MODULE_VERSION 115 vs 121 means your better-sqlite3 database was compiled for your local version of Node.js, but Electron uses its own internal version of Node.js, and they don't match.

Since your project relies on better-sqlite3 (the Local SQLite DB from your UML diagram), we need to rebuild it specifically for Electron's version of Node.

The Fix: Rebuild for Electron
Run this command in your terminal. It uses electron-rebuild (which is already in your scripts) to force the database module to match Electron:

PowerShell
npm run rebuild
If that command fails or you get a "not found" error, run this manual version:

PowerShell
npx electron-rebuild -f -w better-sqlite3
What to do if it still fails:
If the rebuild fails with a "missing C++ compiler" or "Python" error, it's because Windows needs build tools to compile sqlite. In that case, use this shortcut to get a pre-compiled version:

Uninstall the current version:

PowerShell
npm uninstall better-sqlite3
Re-install it:

PowerShell
npm install better-sqlite3 --legacy-peer-deps
Run the rebuild again:

PowerShell
npm run rebuild
Why this happened:
Native modules like better-sqlite3 contain C++ code. Every time you update Electron or Node, the "bridge" between the C++ and Javascript changes (that's the NODE_MODULE_VERSION). Electron is almost always a few versions ahead of the standard Node.js release.

Try npm run rebuild now, and then try npm run dev again. The window should finally pop up!
